## 简短总结

- CRNN 明显更强的类别：`Alarm_bell_ringing`、`Blender`、`Cat`、`Dishes`、`Dog`、`Running_water`、`Vacuum_cleaner`，核心优势体现在类别覆盖更完整、事件边界更稳、复杂场景下不容易直接归零。
- BEATs 相对还能工作的类别：`Frying`、`Electric_shaver_toothbrush`、`Speech`、`Vacuum_cleaner`；这些类别大多与长持续或较强语义背景相关，但更多体现为粗粒度覆盖。
- 最能体现互补性的类别：`Running_water`、`Vacuum_cleaner`、`Frying`。它们提示 BEATs 可能保留长持续声学语义，但需要更好的边界建模、阈值和后处理，才能转化成事件级收益。
- 下一步最值得优先关注的类别：`Dog`、`Cat`、`Dishes`、`Alarm_bell_ringing`、`Running_water`。前四类适合验证“类别塌缩”问题，`Running_water` 适合验证“intersection 还行但 event F1 很差”的边界问题。

- “是否长持续类 / 是否短事件或动物类 / 是否多事件场景易失败” 为基于两份报告中的逐类表现、典型样本分析以及类别本身声学常识做的合理标注，不是严格数据集定义。
- 这张表优先突出实验方向判断：CRNN 更强在类别覆盖、边界和复杂场景；BEATs 更偏向长持续类的粗粒度语义覆盖。
