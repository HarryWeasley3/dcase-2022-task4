# BEATs 后处理搜索简报

## 方法说明

这一步没有重训模型，也没有改 SED 底座。由于现成结果目录只保存了解码后的多阈值 TSV，无法直接继续搜索新的 median filter，因此脚本使用现有 `epoch=27-step=23352.ckpt` 重新做了一次 student 前向，导出 frame-level probability，再进行离线后处理搜索。

搜索分成两部分：1. 统一 `threshold + median` 网格搜索；2. 每个类别分别搜索最优 `threshold + median`，再把这些按类最优参数拼成一个组合模板。PSDS 的 per-class 结果采用“按类阈值模板 + 全局缩放因子”的近似方式计算，因此可用于方向判断，但不应视为正式最终口径。

## 简短结论

- 调后处理后，统一搜索和按类搜索都能带来一定提升；但从总体结果看，提升幅度有限，而且伴随明显的 micro F1 / intersection 代价。
- 这轮粗网格里没有出现“单独搜索后仍严格为 0”的类别，但多数字段的提升依赖极低阈值和大量预测段，说明可分性仍然不足。
- 从本轮搜索看，最受益的类别主要是 `Vacuum_cleaner`, `Blender`, `Running_water`, `Frying`；这些类更像“原本有分数，但边界/稀疏性没调好”。
- 如果 PSDS 和 event-based F1 只能小幅回升，而 `Dog / Dishes / Alarm_bell_ringing / Cat / Blender` 在合理整体 operating point 下依然很差，那么结论会更偏向“后处理不是主矛盾，主矛盾仍是 frozen BEATs 下的类别表征不足与类别塌缩”。

## 搜索设置

- threshold 网格：`[0.01, 0.1, 0.3, 0.5, 0.7, 0.9, 0.99]`（粗网格，用于第一步方向判断）
- median filter 网格：`[1, 7, 15]`
- per-class PSDS 缩放网格：`0.6..1.4`，步长 `0.05`
- 当前 PSDS 是基于粗网格 operating points 的近似结果，适合比较“是否变好”，不宜直接和完整 50-threshold 正式口径横向硬比。
